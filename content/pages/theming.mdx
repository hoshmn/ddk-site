## Theming

Theming and styling components throughout the site is handled using [@material-ui/styles](https://material-ui.com/styles/basics/).

All components in the Hyperobjekt Core have been built so their styles can be overriden within a theme configuration. The theme configuration can be overriden by editing the theme file in `/src/gatsby-theme-hyperobjekt-core/theme.js`.

The theme is a JSON configuration containing values for:

- breakpoints
- color palette
- typography
- layout and spacing
- shadows
- transitions

[Explore the default theme](https://material-ui.com/customization/default-theme/) to see everything that is defined in the theme, and their default values.

**Simple Theme:** `/src/gatsby-theme-hyperobjekt-core/theme.js`  
_redefines the primary color palette used throughout the site to use greys_

```js
const theme = {
  palette: {
    primary: {
      light: "#ccc",
      main: "#999",
      dark: "#666",
    },
  },
}

export default theme
```

### Colors

Colors are defined in the `palette` key of the theme object. There are several palettes within the theme, including:

- primary colors
- secondary colors
- text colors
- background colors

You can specify colors manually, or use one of the palette [provided by Material UI](https://material-ui.com/customization/color/#color-palette).

#### Examples

<br />

**Using a Material UI palette:** `/src/gatsby-theme-hyperobjekt-core/theme.js`  
_redefines the primary color palette to use pink_

```js
import pink from "@material-ui/core/colors/pink"

const theme = {
  palette: {
    primary: pink,
  },
}

export default theme
```

<br />

**Change text and background:** `/src/gatsby-theme-hyperobjekt-core/theme.js`  
_redefines the primary text color and the default background_

```js
const theme = {
  palette: {
    text: {
      primary: "#333",
    },
    background: {
      default: "#eee",
    },
  },
}

export default theme
```

### Breakpoints

Breakpoints are defined in the `breakpoints` key of the theme object.

```jsx
const theme = {
  breakpoints: {
    values: {
      xs: 0,
      sm: 600,
      md: 960,
      lg: 1280,
      xl: 1920,
    },
  },
}
```

The theme provides helper functions to define styles based on the specified breakpoints. These functions include:

- `theme.breakpoints.up(key)`
- `theme.breakpoints.down(key)`
- `theme.breakpoints.only(key)`
- `theme.breakpoints.between(start, end)`

#### Examples

<br />

**Styling using breakpoints:** `/src/gatsby-theme-hyperobjekt-core/theme.js`  
_override H1 font sizes at different breakpoints_

```jsx
const baseTheme = ({ theme }) {
  typography:{
    h1: {
      fontSize: theme.pxToRem(24),
      [theme.breakpoints.up("sm")]: {
        fontSize: theme.pxToRem(28)
      },
      [theme.breakpoints.up("md")]: {
        fontSize: theme.pxToRem(32)
      },
      [theme.breakpoints.up("lg")]: {
        fontSize: theme.pxToRem(40)
      },
      // you can also specify a number for width
      [theme.breakpoints.up(1920)]: {
        fontSize: theme.pxToRem(48)
      },
    }
  }
}
export default baseTheme
```

### Overrides

#### Examples

<br />

**Set a global override:** `/src/gatsby-theme-hyperobjekt-core/theme.js`  
_sets a style for font smoothing on the `<html>` element for the page_

```jsx
const theme = {
  overrides: {
    MuiCssBaseline: {
      "@global": {
        html: {
          WebkitFontSmoothing: "auto",
        },
      },
    },
  },
}

export default theme
```

### Icons

In order to customize icons on the site, you can override the default icons provided by the them using component shadowing.

<br />

**Override Icons:** `/src/gatsby-theme-hyperobjekt-core/icons.js` <br />
_redefines themenu icons with the FontAwesome menu icon_

```js
import { icons } from "gatsby-theme-hyperobjekt-core"
import { FaBars } from "react-icons/fa"

const overrideIcons = {
  ...icons,
  menu: MenuIcon,
}

return overrideIcons
```
